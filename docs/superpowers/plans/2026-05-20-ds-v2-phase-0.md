# DS v2 — Phase 0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Poser l'infrastructure du Design System v2 (tokens, ThemeContext, components/ds/) sans casser aucun écran existant.

**Architecture:** Cohabitation legacy/v2. L'ancien `contexts/ThemeContext.tsx` est renommé en `.legacy.tsx`, le nouveau prend sa place avec une API différente (`colors.fg.primary`, `colors.accent.default`). Patch : force `scheme = 'light'` (pas de dark mode). Les 4 consumers actuels du legacy (App.tsx, 2 screens, 1 component) repointent vers `.legacy`. Aucun écran n'est migré pendant cette phase.

**Tech Stack:** React Native + Expo SDK 54, TypeScript strict, React Context. Dépendances ajoutées : `expo-symbols`, `react-native-gesture-handler`.

**Spec source :** [`docs/superpowers/specs/2026-05-20-design-system-v2-migration-design.md`](../specs/2026-05-20-design-system-v2-migration-design.md) — section "Phase 0"

**Verification model :** Pas de framework de test unitaire dans ce projet. Chaque étape est validée par (a) compilation TypeScript propre (`npx tsc --noEmit`) et (b) au final, build & boot du simulateur iOS (`npm run ios`). Les "tests" sont des smoke tests manuels.

## Task 5: Installer le nouveau ThemeContext (light-only patch)

**Files:**
- Create: `contexts/ThemeContext.tsx` (nouvelle version DS v2 patchée)

- [ ] **Step 5.1: Créer `contexts/ThemeContext.tsx` avec le patch light-only**

Créer le fichier avec ce contenu exact (issu de `.design-system-v2-tmp/contexts/ThemeContext.tsx` mais simplifié pour forcer light) :

```tsx
// ============================================================================
// ZeroGaspy Design System · ThemeContext (v2)
// ============================================================================
// Patch local : scheme forcé en 'light'. Le toggle dark mode n'est pas exposé.
// Les exports darkColors restent disponibles dans tokens/colors.ts pour usage
// futur si on réactive le dark mode.
// ============================================================================

import React, { createContext, useContext, useMemo, ReactNode } from 'react';

import {
  getColors,
  typography,
  space,
  layout,
  elevation,
  radius,
  componentRadius,
  type ColorScheme,
  type SemanticColors,
} from '@/tokens';

interface ThemeContextValue {
  scheme: ColorScheme;
  setSchemeOverride: (s: ColorScheme | null) => void;
  colors: SemanticColors;
  typography: typeof typography;
  space: typeof space;
  layout: typeof layout;
  elevation: typeof elevation;
  radius: typeof radius;
  componentRadius: typeof componentRadius;
}

const ThemeContext = createContext<ThemeContextValue | undefined>(undefined);

interface ThemeProviderProps {
  children: ReactNode;
}

export function ThemeProvider({ children }: ThemeProviderProps) {
  const value = useMemo<ThemeContextValue>(
    () => ({
      scheme: 'light',
      setSchemeOverride: () => {},
      colors: getColors('light'),
      typography,
      space,
      layout,
      elevation,
      radius,
      componentRadius,
    }),
    [],
  );

  return <ThemeContext.Provider value={value}>{children}</ThemeContext.Provider>;
}

export function useTheme(): ThemeContextValue {
  const ctx = useContext(ThemeContext);
  if (!ctx) throw new Error('useTheme must be used within a ThemeProvider');
  return ctx;
}

export function useColors(): SemanticColors {
  return useTheme().colors;
}

export function useColorScheme(): ColorScheme {
  return useTheme().scheme;
}
```

```tsx
function App() {

    <ErrorBoundary>
      <LegacyThemeProvider>
        <AuthProvider>
          <SubscriptionProvider>
            <GamificationProvider>
              <RootNavigator />
            </GamificationProvider>
          </SubscriptionProvider>

```

La remplacer par :

```tsx
function App() {
  return (
    <GestureHandlerRootView style={{ flex: 1 }}>
      <ErrorBoundary>
        <LegacyThemeProvider>
          <DSThemeProvider>
            <AuthProvider>
              <SubscriptionProvider>
                <GamificationProvider>
                  <ToastProvider bottomOffset={49}>
                    <RootNavigator />
                  </ToastProvider>
                </GamificationProvider>
              </SubscriptionProvider>
            </AuthProvider>
          </DSThemeProvider>
        </LegacyThemeProvider>
      </ErrorBoundary>
    </GestureHandlerRootView>
  );
}
```

**Ordre des providers (extérieur → intérieur)** :
1. `GestureHandlerRootView` — wrapper le plus externe (requis par Swipeable)
2. `ErrorBoundary` — capture crashes en dessous
3. `LegacyThemeProvider` — ancien Theme (pour les 3 consumers legacy)
4. `DSThemeProvider` — nouveau Theme DS (pour `ToastProvider` et futurs composants DS)
5. `AuthProvider`, `SubscriptionProvider`, `GamificationProvider` — existants
6. `ToastProvider` — doit être sous `DSThemeProvider`, au-dessus de la navigation
7. `RootNavigator` — contient le `NavigationContainer`
